# 구현 요구사항 정리

## 도메인 요구사항 정리

### 유저(회원, 트레이너)
1. 필요한 정보
   - 아이디
   - 비밀번호
   - 이름
   - 생년월일
   - 성별
   - 권한

### 예약
1. 필요한 정보
   - 어떤 예약 일정인지
   - 누가 예약했는지

### 예약 일정
1. 필요한 정보
   - 예약 날짜
   - 예약 시간
   - 어떤 트레이너의 일정인지

### 요일별 기본 예약 일정
1. 필요한 정보
   - 예약 시간
   - 어떤 트레이너의 기본 예약 일정인지

### 트레이너별 에약 오픈 기간
1. 필요한 정보
   - 몇달의 예약을 받을건지
   - 어떤 트레이너의 정보인지

## 비즈니스 요구사항 정리
1. 회원은 로그인을 할 수 있다.
2. 회원은 특정 트레이너를 선택해서 예약을 할 수 있다.
3. 트레이너는 예약 오픈 기간을 등록할 수 있다.
   - 트레이너는 하나의 예약 오픈 기간만 가질 수 있다.
4. 트레이너는 요일별 예약 기본 일정을 추가할 수 있다.
